Leetcode Easy

O(1): Constant time - The algorithm runs in constant time regardless of input size.

O(log n): Logarithmic time - The classic example is binary search.

O(n): Linear time - The runtime grows linearly with the size of the input.

O(n log n): Linearithmic time - Seen in algorithms like merge sort or quicksort.

O(n2): Quadratic time - Seen in some algorithms with nested loops.

O(n3): Cubic time - Seen in algorithms with three nested loops, and so on for higher powers.

O(2n): Exponential time - For algorithms like recursive calculations of the Fibonacci numbers.

O(n!): Factorial time - For algorithms like the naive traveling salesman problem.
